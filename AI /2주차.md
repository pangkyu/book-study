## 클로드 코드 가이드

https://wikidocs.net/335592

### 설치

탭마다 역할과 로컬 파일 접근 여부가 다릅니다.

chat : 질문, 브레인스토밍, 문서 초안 등의 일반적인 대화이며, 로컬 파일 접근할 수 없습니다.

cowork : 긴 작업을 맡기면 클라우드에서 자율적으로 실행. 앱을 닫아도 계속 작업을 진행합니다. 로컬 파일에는 접근할 수 없습니다.

code : 내 프로젝트 폴더의 파일을 직접 읽고 수정합니다. 변경을 확인하고 승인합니다. 로컬 파일에 직접 접근 가능합니다.

### 대화와 세션

세션을 종료한 뒤에도 이전 대화를 이어서 할 수 있습니다.

1. 마지막 대화 이어가기
   `/exit`으로 세션을 종료한 뒤, `-c`플래그로 다시 시작합니다
   `claude -c`

2. 파일참조(@)
   대화 중에 특정 파일을 언급하려면 `@`를 입력합니다

3. 현재 세션에 이름 붙이기
   대화 중에 `/rename`으로 이름을 붙입니다.

4. 이름으로 세션 재개. 종료 후 이름으로 바로 재개할 수 있습니다.
   `claude -r '버그수정'`

### 슬래시 명령어

`/model` : 모델 변경
`/voice` : 음성 입력 모드
`/resume` : 이전 세션 목록에서 선택하여 재개.
`/clear` : 대화 이력 초기화
`/compact` : 대화 핵심만 남기고 압축

### 되돌리기

클로드가 파일을 수정할 때마다 자동으로 체크포인트를 만듭니다.
결과가 마음에 들지않으면 `esc`를 2번 누릅니다

세션 기록의 삭제 기간도 정할 수 있습니다.

### 모델

높은 모델일수록 토큰을 많이 사용하기 때문에 업무에 따라 조절해서 사용하는 것을 추천합니다

opus, sonnet, haiku

같은 모델이라도 사고 깊이를 조절할 수 있습니다

`/effort [low|medium|high|xhigh|max|auto]`

**ultrathink**
한번만 깊이 생각하게 하고 싶을때 사용합니다. 문장 앞, 뒤, 중간 어디에 있어도 됩니다

### 권한 모드

아래로 갈수록 자욜성이 높습니다.

`plan` : 코드 수정 불가, 분석/계획만 수행
`default` : 파일 읽기는 자동, 수정/명령 실행은 승인 필요
`acceptEdits` : 파일 수정은 자동 허용, 명령 실행은 승인 필요
`auto` : 분류기가 백그라운드에서 안전검사.
`dontask` : /permissions나 설정 파일에서 사전 승인된 것만 허용, 나머지는 자동 거부.
`bypassPermissions` : 모든 작업 자동 실행

**플랜모드**
일반적으로 코드 작성을 시키는 행위가 아닌, 현재 상태를 분석하고 어떤 순서로 무엇을 할지 정리해서 보여줍니다.

해당 방식으로 진행하면 방향 수정 시간을 크게 줄일 수. ㅣㅆ습니다.

**민감 정보 보호**
.env나 비밀번호가 포함된 파일이 있으면 접근을 차단할 수 있습니다.

`.claude/setting.json`에서 deny 규칙을 추가합니다.

```json
{
  "permissions": {
    "deny": [
      "Read(.env)",
      "Read(.env.*)",
      "Read(**/.env)",
      "Read(**/.env.*)",
      "Read(secrets/**)",
      "Edit(.env)",
      "Edit(.env.*)",
      "Edit(**/.env)",
      "Edit(**/.env.*)",
      "Edit(secrets/**)",
      "Bash(cat:*.env*)",
      "Bash(cat:*secrets/*)"
    ]
  }
}
```

### 컨텍스트 관리

클로드는 한번에 처리할 수 있는 대화량에 한계가있습니다. (컨텍스트 윈도우)

`/clear`(새 노트로 시작)와 `/compact`(요약해서 공간확보) 명령어로 컨텍스트 관리를 할 수 있습니다.

컨텍스트가 얼마나 차있는지 확인하려면 `/context`를 입력합니다.
대화 이력, 파일 내용, 명령 실행결과, 로드된 스킬, 시스템 지시, 클로드.md가 모두 포함됩니다

컨텍스트가 95%가 차면 자동압축이되나, 50%정도에서 미리 압축하는 것이 비용이 적습니다. 또한, 컨텍스트가 작을수록 응답 품질이 좋습니다.

클로드 코드는 한 번의 대화를 세션이라고 하는데, 세션을 종료하고 시작하면 백지 상태입니다.

이를 해결하기 위해 `claude.md`를 사용하여 세션이 시작될 때 파일을 컨텍스트에 자동 주입하도록 합니다.

1. init으로 자동 생성
   클로드가 파일구조, package.json, readme 등을 분석하여 claude.md 초안을 생성합니다. 이미 기 파일이 있는 경우 덮어쓰지 않고 개선안을 제안합니다.

2. 직접 작성하기

자동 생성 대신 직접 내용을 지정하면 더 정확한 지침을 줄 수 있습니다.

```md
# 프로젝트 개요

할일 관리 REST API - 사용자별 할일 CRUD 및 카테고리 분류

## 기술 스택

- Node.js 20 + Express
- PostgreSQL + Prisma
- Jest + Supertest (테스트)

## 빌드/테스트 명령어

- 개발 서버: `npm run dev`
- 테스트 전체: `npm test`
- 단일 테스트: `npm test -- --testPathPattern=파일명`
- 린트: `npm run lint`

## 코드 규칙

- require 대신 import/export 사용 (ES Modules)
- 에러 핸들링은 express-async-errors 미들웨어 사용
- 환경변수는 dotenv로 관리, .env 파일은 커밋하지 않음
```

넣어야할 것

- 클로드가 추측할 수 없는 명령어
- 기본값과 다른 코드 스타일 규칙
- 저장소 관례
- 프로젝트 고유 아키텍처 결정
- 비개발 업무의 문서 형식, 톤, 반복 패턴

빼야할 것

- 클로드가 코드를 읽으면 알 수 있는 것
- 언어의 표준 관례
- 자주 바뀌는 정보
- 긴 설명이나 튜토리얼
- 파일별 상세 설명

파일당 200줄 이내를 권장하고, 깃에 커밋되기때문에 API키나 비밀번호는 넣지 않도록 주의합니다
