# 관리

## 코드 정리 구분

코드 정리하는 것은 현미경으로 그림을 보다가 초점을 못추는 것과 비슷합니다.
변경 사항을 나누어 별도의 PR을 만들어야합니다.
순서가 있는 일련의 코드 정리는 1개의 PR로 작성합니다.
동작 변경 역시 별도 PR로 만듭니다.

코드 정리와 동작 변경 사이를 번갈아 가면서 전환할 때마다 새로운 PR을 열어야합니다.

작은 단위로 PR을 작성하는 것을 추천합니다.

## 연쇄적인 정리

코드를 계속 정리하고 싶은 충동을 관리하는 것도 코드 정리의 핵심 기술입니다.
책에서는 아주 작은 단계로 나눠 코드를 정리하는 방식을 고수하면서 실험하는 것입니다.

**보호 구문**
보호 구문을 넣어 코드를 정리하면 조건이 설명하는 도우미로 드러나거나 설명하는 변수 추출을 돕는 혜택이 있습니다.

**안쓰는코드**
사용하지 않는 더미를 제거하면 코드 읽는 순서에 맞춰 정렬하는 방법과 응집도를 높이는 배치가 보입니다.

**대칭으로 맞추기**
같은 코드와 다른 코드를 일치시키다보면 유사한 코드들이 묶여진 순서대로 읽을 수 있습니다.

**읽는 순서**
읽는 순서를 정리하면 대칭으로 맞출 기회가 생깁니다.

**응집도를 높이는 배치**
응집도를 높이는 배치로 함께 묶인 요소는 하위 요소로 추출할 후보가 됩니다.
헬퍼를 만드는 것은 코드 정리를 넘어서지만 정리에 익숙해지고 자신감이 생기면 더 큰 규모로 설계 변경을 할 수 있습니다.

**설명하는 변수**
할당하는 문장에서 좌변이 설명하는 변수라면 그에 대응하는 우변은 설명하는 헬퍼 후보일 수 있습니다.

**설명하는 상수**
설명하는 상수 정리는 응집도를 높이는 배치를 이끕니다.

**명시적인 매개변수**
매개변수를 명확히 드러나게 만들면 매개변수 집합을 묶어 객체로 만들고 코드로 옮길 수 있습니다.

**비슷한 코드끼리**
코드 덩어리 앞에 설명하는 주석을 붙일 수 있습니다.

소프트웨어 개발에서 변경은 가장 많은 비용이 듭니다. 그 중에서도 코드를 이해하는 일에 가장 비용이 많이 듭니다.
따라서 작동하는 코드 구조와 의도에 대해 잘 소통하는 것이 익혀야할 가장 가치있는 기술입니다.

-> 코드 구조를 너무 많이, 너무 빠르게 변경하지 않도록 주의합니다.

## 코드 정리의 일괄 처리량

**충돌**
일괄 처리하는 코드 정리 작업이 많을 수록 통합 과정에서 지연 시간이 길어집니다.
코드 정리 작업이 다른 사람의 진행 중인 작업과 충돌할 간으성도 커집니다.

**상호작용**
다수의 코드 정리를 한번에 처리하다가 우연히 동작 변경을 할 수 있습니다.
코드 정리 사이에 상호작용이 있으면 병합 비용은 급격히 증가합니다.

**추측**
한 번에 처리하는 코드 정리가 많을수록 자연스럽게 더 많은 코드를 정리합니다.

많은 조직에서 하나의 변경 사항을 검토하고 배포하는데 드는 고정 비용은 상당히 많습니다

## 리듬

소프트웨어 설계는 프랙탈이므로 설계는 크게도 작게도 할 수 있습니다
한 번의 코드 정리에 한 시간 이상 걸렸다? -> 원하는 동작 변경을 위해 필요한 최소한의 구조 변경 시기를 놓쳤다는 의미일 수 있습니다

하지만 코드가 너무 엉망이면 몇 시간을 들여서라도 코드 정리를 하는 게 더 유리할 수 있습니다

동작 변경은 코드 안에 뭉쳐서 나타나는 경향이 있습니다.
파레토 법칡에 따르면 80%의 변경 사항이 20%의 파일에서 발생합니다.

## 코드 정리 시점

코드 정리를 먼저하고 동작 변경을 하나요? 아니면 동작 변경을 한 뒤 코드 정리를 할까요?

**코드 정리를 안하는 경우**
가장 그럴듯한 이유는 코드동작이 앞으로 바뀌지 않을 것이라는 믿음입니다.
진짜 변경이 필요없는 시스템이면 고장 안나면 고치치말자 라는 말이 통합니다

**동작 변경 후에 코드정리**

- 방금 고친 코드를 다시 변경할 예정일때
- 지금 정리하는 것이 더 저렴할때
- 코드 정리하는 데 드는 시간이 동작 변경에 드는 시간과 거의 비슷할때

동작 변경에 한 시간이 걸렸다고 가정한다면 한 시간 정도는 코드 정리에 투자하는 것이 합리적입니다

**코드 정리 후에 동작 변경**

- 어떤 코드를 어떻게 정리해야 하는지 알고 있을 때
- 코드 정리를 했을 때 코드 이해가 쉬워지거나 동작 변경이 쉬워지는 즉각적인 효과를 얻을 수 있을 때
