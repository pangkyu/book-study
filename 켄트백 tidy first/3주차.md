## 3주차

### 요소들을 유익하게 관계 맺는 일

토큰 -> 식 -> 문 -> 함수 -> 객체/모듈 -> 시스템

요소에는 경계가 있습니다. 그래서 어디서 시작하고 끝나는 지 알 수 있습니다.

또한 요소는 하위 요소를 포함합니다.

**관계맺기**
요소들은 서로 관계를 지닌 존재입니다. (하나의 함수가 다른 함수를 호출)

소프트웨어 설계에서 관계는 아래가 있습니다.

- 호출
- 발행(publish)
- 대기(listen)
- 참조(변수의 값을 가져오기 )

### 구조와 동작

소프트웨어는 2가지 방식으로 가치를 만듭니다.

- 현재 소프트웨어가 하는일
- 미래에 새로운 일을 시킬 수 있는 가능성

코드 정리는 아래의 식이 성립될 때 바로 코드 정리하는 것을 추천합니다.
`비용(코드 정리) + 비용(코드 정리 후 동작 변경) < 비용(바로 동작 변경)`
